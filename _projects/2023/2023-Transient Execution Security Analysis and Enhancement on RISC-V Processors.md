abstract: >-
  1) Analyzed Spectre v1 timing behavior on RISC-V processors and
  systematically characterized transient execution security
  properties.<br/>
  2) Proposed Flush Key Load (FKL), a novel detection metric for
  identifying security-relevant cache access patterns associated
  with transient execution; validated its effectiveness on gem5
  and Chipyard.<br/>
  3) Designed a dual-layer mitigation strategy using FKL to
  trigger coordinated hardware-software response; benchmark
  results demonstrated effective timing leakage suppression
  with only ~2% performance overhead.<br/>
